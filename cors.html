CORS / CORP

Cross-Origin Resource Sharing - междоменный обмен ресурсами. Origin (источник) - домен (доменое имя), с которого производится запрос. Cross-Origin - ситуация, когда домен, отсылающий запрос, и запрашиваемый домен разные. Ситуация возникает когда а) API общедоступный, например, служба новостей, погоды, курсов валют и т.п. б) Система едина, но распределенная. Например, запрос к API производится с мобильного устройства

Cross-Origin Resource Policy (CORP) - принцип, согласно которому только внутридоменные запросы проходят без дополнительных ограничений.

Для проверки и настройки CORS / CORP нужно проводить запросы из разных доменов. Простейший способ - открыть страницу как отдельный файл, а не через localhost

Без дополнительных действий кросс-доменный запрос блокируется браузером, если в ответе сервера нет заголовка 'Access-Control-Allow-Origin'

Для запросов, кроме GET и POST, в кросс-доменном варианте предварительно посылается запрос OPTIONS. Сервер должен подтвердить тот факт, что он вообще допускает запросы другого типа. Если сервер не отвечает на этот запрос или отвечает не должным образом, то запрос, например PUT, даже не будет отправляться клиентом.
